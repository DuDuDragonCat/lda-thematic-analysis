# LDA 中文主題模型分析工具

本工具為以下學術論文所提出研究方法的 GUI 實作：

> Shaw, F. F.-T., **Yu, Y.-T.**, Yang, L.-X., Wong, S.-H., & Hsu, W.-Y. (2025). 潛在狄利克雷分布技術於文本主題分析之應用：以三位自殺死亡個案的協談內容為例。*中華心理學刊，67*(4)，335–360。https://doi.org/10.6129/CJP.202512_67(4).0003

論文以三位自殺死亡個案的協談逐字稿為語料，將 LDA 應用於繁體中文質性文本的主題分析，並結合四項指標（Arun 2010、Cao 2009、Deveaud 2014、Griffiths & Steyvers 2004）選擇最佳主題數，再透過「段落主題命名程序」保留原始語境以提升詮釋深度。本工具將該完整流程包裝為四步驟 GUI，使研究者無需撰寫程式碼即可重現論文方法。

---

基於潛在狄利克雷分布（Latent Dirichlet Allocation）的繁體中文文本主題分析 GUI 工具，整合 [CKIP 中文斷詞](https://github.com/ckiplab/ckip-transformers)與 [tomotopy](https://github.com/bab2min/tomotopy) 進行 LDA 模型適配，提供四步驟完整分析流程。

## 功能概覽

| 步驟 | 功能 | 說明 |
|------|------|------|
| 步驟 1 | 下載斷詞模型 | 自動從 HuggingFace 下載 CKIP BERT 模型 |
| 步驟 2 | 執行中文斷詞 | 對 `.xlsx` 的 `content` 欄進行繁體中文斷詞 |
| 步驟 3 | 選擇最佳 K | 計算四項 LDA 指標，繪製折線圖輔助選定主題數 |
| 步驟 4 | LDA 適配 | 以選定的 K 值適配模型，輸出三份結果檔案 |

## 安裝與執行

### 環境需求

- Python 3.12+
- 建議使用 [uv](https://github.com/astral-sh/uv) 管理虛擬環境

### 安裝依賴

```bash
uv sync
```

或使用 pip：

```bash
pip install -e .
```

### 啟動應用程式

```bash
python main.py
```

## 資料格式要求

輸入資料為 **Excel 檔案（`.xlsx`）**，需符合以下規格：

| 欄位 | 必要 | 說明 |
|------|------|------|
| `id` | 建議 | 段落或句子的識別編號 |
| `content` | **必要** | 中文文本內容（斷詞對象） |
| 其他欄位 | 可選 | 分析時不影響結果 |

**資料單位**：每一列（row）代表一段落或一句話。

**範例：**

| id | content |
|----|---------|
| 1 | 我覺得最近情緒很不穩定，常常… |
| 2 | 那你平常有沒有想到什麼方法來… |
| 3 | 我有試過跟家人說，但他們不太… |

## 操作步驟

### 步驟 1：下載 CKIP 中文斷詞模型

CKIP（中研院中文斷詞系統）使用 BERT 深度學習模型進行繁體中文斷詞，**模型只需下載一次**。

1. 按「瀏覽」選取一個資料夾作為模型存放位置（建議固定路徑，如 `~/models`）
2. 按「開始下載」，程式會自動下載：
   - `ckiplab/bert-base-chinese-ws`（斷詞模型）
   - `ckiplab/bert-base-chinese-pos`（詞性標記模型）
3. 等待下方文字框顯示「=== 全部下載完成 ===」

> 首次下載約需 1–2 GB，請確保網路連線穩定且可連接 HuggingFace。

---

### 步驟 2：執行中文斷詞

1. 確認模型資料夾路徑（與步驟 1 相同）
2. 按「載入模型」，等待狀態顯示綠色**「已載入」**（首次約需 30 秒至 2 分鐘）
3. 選擇**輸入資料夾**（含 `.xlsx` 檔案）
4. 選擇**輸出資料夾**（不可與輸入資料夾相同）
5. 按「執行斷詞」，等待 Log 顯示完成訊息

#### 輸出欄位說明

| 欄位 | 說明 |
|------|------|
| `sent_seq` | 原始第幾筆對話（row 編號） |
| `word` | CKIP 斷詞後的單一詞彙 |
| `pos` | 詞性標註（CKIP POS Tags） |
| `seq` | 該詞在句子中的位置序號 |

**輸出範例**（輸入：「我真的不想活了。沒有人在乎我。」）：

| sent_seq | word | pos | seq |
|----------|------|-----|-----|
| 1 | 我 | Nh | 0 |
| 1 | 真的 | D | 1 |
| 1 | 不 | D | 2 |
| 1 | 想 | VE | 3 |
| 2 | 沒有 | VJ | 0 |
| 2 | 人 | Na | 1 |
| 2 | 在乎 | VK | 2 |

---

### 步驟 3：計算指標，選擇最佳主題數 K

#### 參數設定

| 參數 | 預設值 | 說明 |
|------|--------|------|
| SEED | 777 | 隨機種子，確保結果可重現 |
| Burn In | 500 | MCMC 暖機次數，讓模型穩定 |
| Iteration | 1000 | 抽樣迭代次數，越大越穩定但越慢 |
| Thin Interval | 100 | 間隔多少次抽樣保留一次，降低自相關性 |
| TopicK (F/T/B) | 2 / 10 / 2 | 搜尋最佳 K 的範圍（最小值 / 最大值 / 間隔） |

#### 評估指標

| 指標 | 方向 | 說明 |
|------|------|------|
| Griffiths 2004 | 越高越好 ↑ | 模型對語料庫的解釋能力（對數概似） |
| Cao 2009 | 越低越好 ↓ | 主題間相似程度；越低代表主題越獨立 |
| Arun 2010 | 越低越好 ↓ | 文章主題分配與詞彙分配的對稱 KL 散度 |
| Deveaud 2014 | 越高越好 ↑ | 主題間平均 Jensen-Shannon 散度；越高代表主題越多樣 |

執行完成後產生 `LDA_Metrics.xlsx` 及指標折線圖（`.png`）。觀察四個指標的**公約數**（同時達到最佳值的 K）作為最佳主題數。

---

### 步驟 4：LDA 適配與主題分配

1. 選擇「輸入資料夾」（步驟 2 的斷詞結果）
2. 選擇「輸出資料夾」
3. 填入步驟 3 建議的 K 值
4. 按「LDA 模型適配」，等待完成

#### 輸出檔案說明

**`TopicPart_K{n}.xlsx`** — 段落主題分布

| 欄位 | 說明 |
|------|------|
| `document` | 檔名_row@詞位置 |
| `topic_1` … `topic_n` | 各主題的機率值（加總 ≈ 1） |
| `raw_text` | 對應的原始文字片段 |

**`WordTopic_K{n}.xlsx`** — 詞彙-主題分布

| 欄位 | 說明 |
|------|------|
| `topic` | 主題編號（從 0 起） |
| `word` | 代表性詞彙 |
| `log_probability` | 機率取自然對數（負值） |
| `probability` | 詞彙在該主題的機率值 |

**`TopicDoc_K{n}.xlsx`** — 文章主題分配

每篇文章被指派至機率最高的主題，含主題機率分布及最佳主題標籤。

> `{n}` 為選擇的 K 值，例如 K=4 時檔名為 `TopicPart_K4.xlsx`。

## 完整操作流程

```
原始資料 (.xlsx)
    │
    ▼ 步驟 1
下載 CKIP BERT 模型（一次性，約 1~2 GB）
    │
    ▼ 步驟 2
執行中文斷詞 → 斷詞結果 (.xlsx)
    │
    ├──▶ 步驟 3：計算 4 種指標 → 折線圖 → 選定 K 值
    │
    └──▶ 步驟 4：輸入 K 值，適配 LDA 模型
              │
              ├── TopicPart_K{n}.xlsx（段落主題分布）
              ├── WordTopic_K{n}.xlsx（詞彙-主題分布）
              └── TopicDoc_K{n}.xlsx（文章主題分配）
```

## 常見問題

**Q：模型下載很慢或失敗怎麼辦？**
需要穩定的網路連線並能連接 HuggingFace。若速度緩慢可嘗試使用 VPN 或等待後重試。中途失敗可以重新按「開始下載」繼續。

**Q：步驟 2「執行斷詞」按鈕是灰色的？**
需確認三個條件都滿足：① 模型已成功載入（狀態顯示綠色「已載入」）② 已選擇輸入資料夾 ③ 已選擇輸出資料夾。

**Q：步驟 3 要選哪個 K 值？**
觀察指標折線圖，找尋「肘部」（Elbow）位置。Griffiths／Deveaud 取曲線上升趨緩點；Cao／Arun 取曲線下降趨緩點。也可結合研究目的主觀判斷。

**Q：執行時間很長正常嗎？**
是正常現象。步驟 2 斷詞依資料量約需數分鐘至數十分鐘；步驟 3/4 的 LDA 計算會依 K 值範圍和 Iteration 數目而不同，請耐心等待。
